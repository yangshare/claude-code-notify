## 新增需求

### 需求：Windows 原生通知
系统必须使用 Windows 10/11 原生通知 API (Windows Runtime) 显示通知卡片。

#### 场景：在 Windows 10 上显示通知
- **当** 系统检测到 Windows 10 版本 1607 或更高
- **那么** 必须使用 Windows.UI.Notifications API 显示 Toast 通知

#### 场景：通知符合 Fluent Design
- **当** 显示通知卡片
- **那么** 必须符合 Windows 11 Fluent Design 设计规范

### 需求：状态感知图标
系统必须根据任务状态显示不同的图标：
- 成功 (success)：绿色对勾或自定义图标
- 失败 (error)：红色警告或自定义图标
- 等待/进行中 (pending)：黄色沙漏或自定义图标

#### 场景：显示成功状态图标
- **当** 通知状态为 success
- **那么** 通知卡片应显示绿色对勾图标

#### 场景：显示失败状态图标
- **当** 通知状态为 error
- **那么** 通知卡片应显示红色警告图标

#### 场景：使用自定义图标
- **当** 配置文件指定了自定义图标路径
- **那么** 必须加载并显示自定义图标

### 需求：富文本通知内容
通知卡片必须支持以下内容：
- 标题：显示任务类型（如 "Build Task"）
- 摘要：显示耗时、错误码或关键日志（最多两行）
- 进度条（可选）：如果 Hook 提供进度信息

#### 场景：显示成功通知
- **当** 任务成功完成，耗时 15 秒
- **那么** 通知应显示标题 "Task Completed"，内容 "npm test completed in 15s"

#### 场景：显示失败通知
- **当** 任务失败，错误码为 1
- **那么** 通知应显示标题 "Task Failed"，内容包含错误信息和命令

### 需求：通知交互功能
通知卡片必须支持以下交互：
- 点击通知主体：激活并置顶 Claude Code 所在的终端窗口
- 操作按钮：[查看日志]、[重试]、[忽略]

#### 场景：点击通知激活终端
- **当** 用户点击通知卡片主体
- **那么** 必须激活 Claude Code 的终端窗口并将其置顶

#### 场景：点击查看日志按钮
- **当** 用户点击 [查看日志] 按钮
- **那么** 应打开日志文件所在的文件夹

### 需求：声音反馈
系统必须支持声音通知功能：
- 系统提示音：使用 Windows 默认通知声音
- 自定义音效：支持用户指定 .wav 文件
- 静音模式：支持一键静音或根据专注助手状态自动静音

#### 场景：播放成功音效
- **当** 任务成功完成且 `sound_enabled=true`
- **那么** 应播放配置的成功音效

#### 场景：专注助手模式自动静音
- **当** Windows 专注助手已开启且 `focus_assistant_mode=respect`
- **那么** 必须静音，不播放任何声音

### 需求：通知持久化
通知必须在 Windows 通知中心保留历史记录。

#### 场景：通知保留在通知中心
- **当** 用户关闭通知卡片或通知超时消失
- **那么** 通知必须保留在 Windows 通知中心的历史记录中
