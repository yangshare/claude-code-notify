## 新增需求

### 需求：交互式配置向导
系统必须提供交互式终端界面 (TUI)，引导用户完成配置设置。

#### 场景：启动配置向导
- **当** 用户执行 `ccn init`
- **那么** 必须显示现代化的终端交互界面，提供选项菜单

#### 场景：配置通知图标
- **当** 向导显示图标配置选项
- **那么** 必须提供以下选择：
  - [1] 使用默认图标
  - [2] 选择自定义图标路径
  - [3] 跳过（使用当前设置）

#### 场景：配置通知声音
- **当** 向导显示声音配置选项
- **那么** 必须提供以下选择：
  - [1] 启用系统音效
  - [2] 选择自定义 .wav 文件
  - [3] 禁用声音

#### 场景：配置通知阈值
- **当** 向导显示阈值配置选项
- **那么** 必须提示用户输入最小通知时长（秒），默认值为 10

### 需求：配置预览和确认
向导必须在用户完成配置后显示预览，并要求确认。

#### 场景：显示配置预览
- **当** 用户完成所有配置选项
- **那么** 必须显示完整的配置摘要，包括所有选项的当前值

#### 场景：用户确认保存
- **当** 用户在预览界面选择确认
- **那么** 必须将配置保存到配置文件并显示成功消息

#### 场景：用户取消配置
- **当** 用户在预览界面选择取消
- **那么** 必须放弃更改并退出向导

### 需求：配置验证
向导必须验证用户输入的合法性。

#### 场景：验证图标路径
- **当** 用户输入自定义图标路径
- **那么** 必须验证文件存在且是有效的图片格式（.png、.ico、.jpg）

#### 场景：验证声音文件
- **当** 用户输入自定义声音文件路径
- **那么** 必须验证文件存在且是有效的 .wav 格式

#### 场景：验证阈值输入
- **当** 用户输入最小阈值
- **那么** 必须验证是正整数且在合理范围内（1-3600 秒）

### 需求：TUI 界面要求
配置向导必须提供良好的用户体验。

#### 场景：支持键盘导航
- **当** 向导显示选项菜单
- **那么** 必须支持使用方向键和回车键进行选择

#### 场景：显示帮助信息
- **当** 用户在每个配置页面按 F1 或 ? 键
- **那么** 必须显示该选项的详细说明和推荐设置

#### 场景：支持随时退出
- **当** 用户在任何页面按 Esc 或 Ctrl+C
- **那么** 必须提示是否确认退出，未保存的更改将被丢弃
